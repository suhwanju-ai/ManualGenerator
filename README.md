# Manual Capture — 웹앱 사용 매뉴얼 자동 생성 (Chrome 확장 스타터)

웹앱을 평소처럼 클릭하며 따라가면, 각 단계마다 스크린샷이 찍히고 Claude가 단계별
설명을 작성해 자체 완결형 HTML 매뉴얼로 내보냅니다. 비개발자도 녹화 버튼만 누르면 됩니다.

## 설치 (개발자용, 1회)

1. Chrome 주소창에 `chrome://extensions` 입력
2. 우측 상단 **개발자 모드** 켜기
3. **압축해제된 확장 프로그램을 로드** 클릭 → 이 폴더 선택
4. 툴바의 확장 아이콘을 클릭하면 사이드패널이 열립니다.

## 사용

1. 사이드패널 **설정**에서 Claude API 키 입력 (기기에만 저장됨)
2. 문서로 만들 웹앱 탭을 연 뒤 **녹화 시작**
3. 해당 Page에서 F5 click후 매뉴얼에 담을 순서대로 화면을 클릭/입력 → 단계가 자동으로 쌓임
4. **녹화 정지** → **설명 생성** (Claude가 각 스크린샷을 보고 설명 작성)
5. 필요하면 설명을 직접 수정 → **HTML 내보내기**

## 구조

| 파일 | 역할 |
| --- | --- |
| `manifest.json` | MV3 설정·권한 |
| `content.js` | 클릭/입력 감지, 요소 빨간 박스 강조, 컨텍스트 수집 |
| `background.js` | 단계마다 보이는 탭 캡처 후 저장 (휘발성 워커 → storage 보관) |
| `sidepanel.html/js` | 녹화 토글·단계 편집·설명 생성·HTML 내보내기 |

## 알려진 한계 (스타터 기준)

- `captureVisibleTab`은 **보이는 영역만** 캡처합니다. 풀페이지는 스크롤+스티칭 필요.
- 클릭 즉시 페이지가 이동하면 캡처가 다음 화면을 잡을 수 있습니다(드묾).
- cross-origin iframe 내부는 캡처/접근 제한.
- API 키가 확장에 저장되므로, 배포 범위가 넓어지면 **프록시 서버**를 두는 것을 권장.

## 변경 이력

### v0.2.0 (코드 리뷰 반영)
- **[보안/Critical]** 입력 단계에서 `el.value` 기록 제거 — 비밀번호·민감 필드 값이 저장/전송되지 않음. 민감 필드(password, email, tel, card/token 등 이름 힌트 포함)는 스크린샷도 생략.
- **[논리/High]** 단계 저장 race condition 수정 — background에 Promise 직렬화 큐 도입(빠른 연속 클릭 시 단계 유실 해결).
- **[논리/High]** `captureVisibleTab` 호출 간 650ms 최소 간격 스로틀 — 초당 캡처 제한 예외 방지.
- **[논리/Medium]** API 429/5xx 지수 백오프 재시도(최대 3회) 추가.
- **[성능/Medium]** 설명 생성을 동시 3개 제한 병렬 처리(N단계 기준 약 1/3 시간). 사이드패널 storage 쓰기도 큐로 직렬화.
- **[성능/Medium]** 녹화 중 단계 리스트 재렌더 200ms 디바운스.
- **[가독성/Low]** `step.action` 방어적 이스케이프, 매직 넘버 상수화.
