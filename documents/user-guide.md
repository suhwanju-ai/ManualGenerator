# 프로그램 사용 설명서 — Manual Capture

웹앱을 평소처럼 클릭하며 따라가면 각 단계의 스크린샷이 찍히고, AI가 단계별 설명을 작성해 자체 완결형 HTML 매뉴얼로 내보내는 Chrome 확장입니다.

---

## 1. 설치 가이드

### 환경 요구사항
- Google Chrome (또는 Chromium 계열, Manifest V3 + Side Panel 지원 버전)
- AI 설명 생성을 위한 API 키(또는 로컬 비전 모델 서버) — 선택한 제공자에 따라

### 설치 단계 (개발자 모드, 1회)
1. Chrome 주소창에 입력: `chrome://extensions`
2. 우측 상단 **개발자 모드** 토글 켜기
3. **압축해제된 확장 프로그램을 로드** 클릭
4. 이 저장소의 `manual-capture-extension/` 폴더 선택
5. 툴바의 확장 아이콘 클릭 → 사이드패널이 열립니다

> 파일을 수정한 경우, 확장 카드의 새로고침 아이콘을 눌러 재로드하세요.

---

## 2. 초기 설정

사이드패널의 **설정** 패널을 펼쳐 입력합니다. 모든 값은 이 기기에만 저장됩니다.

| 설정 | 설명 |
|---|---|
| AI 제공자 | `Claude` / `OpenRouter` / `OpenAI 호환 커스텀` 중 택1 |
| API 키 | 제공자 키. 커스텀 로컬 서버는 비워도 됨 |
| API Base URL | (커스텀만) `/chat/completions` 앞까지. 예: `http://localhost:11434/v1` |
| 모델 | (OpenRouter/커스텀) 비전 지원 모델명. OpenRouter는 비우면 기본값 사용 |
| 문서 제목 | 내보낼 매뉴얼 제목 (비우면 "사용 매뉴얼") |
| 대상 독자 | (선택) AI 설명 톤 조정용. 예: "처음 사용하는 일반 사용자" |

제공자별 필수 입력:
- **Claude**: API 키
- **OpenRouter**: API 키 (모델은 선택)
- **커스텀**: Base URL + 모델 (키는 선택)

---

## 3. 사용 방법 (기본 흐름)

```
[설정 입력] → [녹화 시작] → [웹앱 클릭/입력] → [녹화 정지]
   → [설명 생성] → (필요시 설명 수정) → [HTML 내보내기]
```

1. 매뉴얼로 만들 웹앱 탭을 엽니다.
2. **● 녹화 시작** 클릭. 상태가 "녹화 중 — 앱을 클릭하세요"로 바뀝니다.
3. 매뉴얼에 담을 순서대로 화면을 클릭/입력합니다. 클릭한 요소가 빨간 박스로 강조되고 단계가 자동으로 쌓입니다.
4. **■ 녹화 정지**.
5. **설명 생성** 클릭 → AI가 각 스크린샷을 보고 단계 설명을 작성합니다(진행 바 표시).
6. 설명이 어색하면 단계 카드의 텍스트를 직접 수정하거나 **삭제**할 수 있습니다.
7. **HTML 내보내기** → 단일 HTML 파일이 다운로드됩니다.

> **수동 캡처**: 녹화 중에는 녹화 버튼 아래에 **📷 현재 화면 캡처** 버튼이 나타납니다. 클릭/입력 없이 지금 보이는 화면을 그대로 한 단계로 추가합니다. 스크롤을 내린 결과 화면, 로딩이 끝난 상태, 알림 팝업 등 자동 캡처로는 잡기 어려운 장면을 매뉴얼에 넣을 때 사용하세요. 추가된 단계도 다른 단계와 똑같이 설명 생성·편집·내보내기가 됩니다.

### UI 구성 (사이드패널)

```
┌─────────────────────────────┐
│ Manual Capture              │
│ [ ● 녹화 시작 ]             │  ← 녹화 토글
│  ● 대기 중                  │  ← 상태 표시
│ [설명 생성] [HTML 내보내기] │
│ ▸ 설정                       │  ← 제공자/키/제목/독자
│ ┌─────────────────────────┐ │
│ │ 1  클릭 · "저장"        │ │  ← 단계 카드 (스크린샷 + 설명)
│ │ [스크린샷]              │ │
│ │ 설명 텍스트(편집 가능)  │ │
│ │                  [삭제] │ │
│ └─────────────────────────┘ │
└─────────────────────────────┘
```

## 4. 민감 정보 보호

다음 입력은 값이 기록되지 않으며 스크린샷도 생략됩니다(라벨은 `[입력값 비공개]`):
- input type: `password`, `email`, `tel`, `number`, `hidden`
- name/id/autocomplete에 `pass`, `pwd`, `secret`, `token`, `card`, `cvv`, `ssn`, `주민`, `비밀` 포함 시

비밀번호 입력 화면 등을 녹화해도 안전하게 단계 메타데이터만 남습니다.

## 5. 관리 기능

| 기능 | 위치 | 동작 |
|---|---|---|
| 기록 전체 지우기 | 설정 패널 하단 | 확인 후 모든 단계 삭제(`steps` 초기화) |
| 단계 개별 삭제 | 각 단계 카드 [삭제] | 해당 단계 제거 후 순번 재계산 |
| 설명 직접 편집 | 단계 카드 텍스트 영역 | 포커스 해제 시 자동 저장 |
| 제공자 전환 | 설정 패널 드롭다운 | 제공자별 키/모델 분리 유지 |

---

## 6. 운영 가이드

- **데이터 위치**: 모든 단계·설정은 브라우저의 `chrome.storage.local`에 저장됩니다(서버 전송 없음).
- **로그/디버깅**: 확장 카드 → "서비스 워커" 링크로 background 콘솔, 사이드패널 우클릭 → 검사로 sidepanel 콘솔 확인.
- **캡처 실패 경고**: `chrome://` 등 캡처 불가 페이지에서는 콘솔에 경고가 남고 해당 단계는 텍스트만 기록됩니다.
- **백업**: 생성한 HTML 파일 자체가 산출물 백업입니다(이미지 인라인, 오프라인 열람 가능).

---

## 7. 업데이트 / 패키징

1. 코드 수정 후 `chrome://extensions`에서 확장 카드 새로고침.
2. 배포용 `.crx` 생성: `manifest.json`의 `version`을 올린 뒤, Chrome DevTools → Extensions → Pack extension(`manual-capture-extension.pem`을 개인키로 지정).

---

## 8. 문제 해결

| 증상 | 원인 / 해결 |
|---|---|
| 단계가 안 쌓임 | 녹화가 꺼져 있음 → "녹화 시작" 확인. 페이지 로드 후 클릭 |
| 스크린샷이 비어 있음 | 민감 입력이거나 캡처 불가 페이지. 보이는 영역만 캡처됨 |
| 설명 생성 실패 토스트 | 키/모델 확인. 429/5xx는 자동 재시도, 반복 시 잠시 후 재시도 |
| 엉뚱한 화면 캡처 | 클릭 직후 페이지 이동 시 발생 가능(드묾) |
| iframe 내부 미캡처 | cross-origin iframe은 접근 불가 |

추가 사용 시나리오·API 관련 FAQ는 → [API 사용자 가이드](api-user-guide.md).
